# PRD Template

## PRD

问题陈述：
从用户视角描述要解决的问题、当前痛点、影响对象和为什么现在需要解决。

解决方案：
描述期望能力、用户可见行为、关键流程和成功状态。保持产品语义清晰，不把实现步骤伪装成需求。

用户故事：
按 actor、能力、收益列出主要用户故事，覆盖核心路径、边界状态和失败状态。

实现决策：
记录已明确的模块、接口、数据流、交互、约束和技术澄清。避免写死容易过期的文件路径；如果原型产出的状态机、schema 或类型片段比文字更准确，可以摘录决策相关部分并说明来源。

测试决策：
说明哪些外部可观察行为需要验证，优先测试公共接口和用户可见结果，不测试实现细节。记录相似测试、手动验证和风险场景。

非目标：
明确本 PRD 不覆盖的范围、暂不处理的边界和不应顺手扩大的工作。

补充说明：
记录假设、待确认问题、后续拆分建议和任何会影响执行顺序的依赖。
